# Code of Conduct

## Our standard

Contributors must communicate respectfully, welcome good-faith questions, critique work rather than people, respect privacy, and avoid harassment, threats, discrimination, sexualized conduct, doxxing, and disclosure of credentials or private vulnerability details.

## Scope

This standard applies in repository issues, pull requests, reviews, discussions, project events, and private project communications, and when representing the project elsewhere.

## Enforcement

Maintainers may edit or remove contributions, issue warnings, temporarily restrict participation, or permanently ban participants when conduct threatens people or the project. Enforcement decisions should be proportionate, documented privately, and avoid conflicts of interest.

## Reporting

Do not publish conduct reports containing private information. The repository owner must designate and publish a private conduct-reporting route before community launch. No contact is invented here; the missing owner-designated route is a launch blocker. Security vulnerabilities should follow `SECURITY.md`, not this process.
